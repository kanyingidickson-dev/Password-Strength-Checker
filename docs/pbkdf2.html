PBKDF2 key derivation

Derive a key from a password locally (SHA-256). Output is hex.

Parameters

PBKDF2 turns a password into a cryptographic key by doing many rounds of hashing.
Salt should be unique per password. It prevents rainbow-table attacks.
Higher is slower but harder to brute-force. 200k is a reasonable starting point.
32 bytes = 256-bit key (common for AES-256).
